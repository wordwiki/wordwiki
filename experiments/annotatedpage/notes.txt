** Code notes
- you are using python function variable and function naming conventions (I know I was as well!)
- we are building up lots of event listeners - each click adds another e.preventDefault(),
  and each popoup adds a new dragstart and dragend (all of which are firing).
- each of these dragstart/dragend pairs are closures that are capturing different tooltipYOffeset etc
  pairs and doing arithmetic with variables that may change on page scroll - I can't reproduce a glitch,
  but is certainly doing something complicated.
- I don't understand the e.pageX == 0 || e.pageY == 0 bit?
- capturing the 'annotatedPage' inside the 'dragend' closure is not ideal.  In practice it will not
  matter - but it is a style that should be avoided (Capturing Dom nodes in a JS closure is potentially
  making GC problems).
- maybe rename 'do_nothing()' to stopPropagation? or something?
